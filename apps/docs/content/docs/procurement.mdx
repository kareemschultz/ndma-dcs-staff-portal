# Procurement

The Procurement module manages Purchase Requisitions (PRs) from creation through to delivery.

## PR lifecycle

`draft` → `submitted` → `approved` / `rejected` → `ordered` → `received`

## Creating a PR

1. Go to **Procurement** → **New Requisition**
2. Add line items (description, quantity, estimated cost, vendor)
3. Set priority and justification
4. Submit for approval

## Approvals

Managers review submitted PRs and either approve or reject with a reason. Approved PRs move to procurement for ordering.

## Tracking

Track when items are ordered and received. The system maintains a full history of all PRs for audit purposes.
