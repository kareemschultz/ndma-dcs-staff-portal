# Procurement

The Procurement module manages Purchase Requisitions (PRs) from initial request through to delivery. It provides a structured, auditable alternative to email-based purchase requests.

## Creating a PR

1. Go to **Procurement** â†’ **New Requisition**
2. Set the title, priority, and justification (why this purchase is needed)
3. Add **line items** â€” one row per product or service:
   - Description
   - Quantity
   - Estimated unit cost (GHS)
   - Vendor name (where known)
4. Save as `draft` to continue editing, or **Submit** immediately for review

## Line items

A PR can have multiple line items. The total cost shown on the PR is the sum of all line item estimates. Line items can be updated while the PR is in `draft` status.

## Approvals

Managers (`manager`, `hrAdminOps`, or `admin` role) review submitted PRs. They can:
- **Approve** â€” moves the PR to `approved` and notifies the requester
- **Reject** â€” requires a reason; the requester is notified and can revise

All approval decisions are recorded in the PR's history and in the audit log.

## After approval

Once approved, the procurement team or admin:
1. Orders the items and clicks **Mark as Ordered** â€” sets status to `ordered`
2. When goods arrive, clicks **Mark as Received** â€” sets status to `received`

## My Requests

The **My Requests** tab shows all PRs created by the currently logged-in user, regardless of status. Use this to track the progress of your own purchase requests without searching the full list.

## Pending Approvals

Managers see a **Pending Approvals** tab listing all submitted PRs that require their review.

## Stats

The procurement stats panel shows:
- Total PRs by status
- Total value of approved PRs
- PRs by priority breakdown
- Average time from submission to approval
