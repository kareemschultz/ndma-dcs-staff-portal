# On-Call Roster

The Roster module manages who is on-call for each week.

## Viewing the current roster

The **Current Roster** shows who is on-call right now and for the current week, including their roles and contact details.

## Roles

| Role | Responsibility |
|------|---------------|
| **Primary** | First point of contact for incidents |
| **Secondary** | Backup if primary is unavailable |
| **Manager on Call** | Escalation point for P1/P2 incidents |

## Publishing a schedule

1. Go to **Roster Planner**
2. Assign engineers to each role for the week
3. The system validates all required roles are filled
4. Click **Publish** to make the schedule live

## Swaps

Engineers can request a swap if they are unavailable. The other engineer must accept before the swap takes effect.

## History

**Roster History** shows all past on-call assignments for audit and reporting.
