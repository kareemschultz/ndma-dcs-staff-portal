# Incident Management

The Incidents module tracks all service disruptions and outages.

## Declaring an incident

1. Go to **Incidents**
2. Click **Declare Incident**
3. Set severity, affected services, and description
4. Add responders from the on-call roster

## Severity levels

| Severity | Description |
|---------|------------|
| **P1 — Critical** | Complete service outage, data loss |
| **P2 — High** | Major feature degraded |
| **P3 — Medium** | Partial degradation, workaround available |
| **P4 — Low** | Minor issue, no user impact |

## Timeline

Add timeline entries as the incident progresses. Every update, decision, and action should be recorded.

## Post-Incident Review (PIR)

Once an incident is resolved, create a PIR documenting:
- Root cause
- Impact assessment
- Timeline of events
- Action items to prevent recurrence
