# Work Management

The Work module tracks all tasks and projects for the DCS team.

## Creating a work item

1. Navigate to **Work Register**
2. Click **New Work Item**
3. Fill in the title, type, priority, and assignee
4. Set a deadline (optional but recommended)

## Work item types

- **Task** — standalone piece of work
- **Bug** — defect to fix
- **Feature** — new capability to build
- **Maintenance** — routine upkeep
- **Investigation** — research or diagnosis

## Statuses

`todo` → `in_progress` → `done`

Items can also be marked `cancelled` or put `on_hold`.

## Weekly updates

Engineers add a weekly update each Friday to record progress. This replaces the weekly status spreadsheet.

## Overdue items

Items past their deadline appear in the **Overdue** view. The system sends a notification to the assignee when a deadline is missed.

## Views

The work register supports multiple views:
- **List** — default table view
- **Kanban** — grouped by status
- **Grid** — card layout
- **Calendar** — items on a monthly calendar
- **Analytics** — bar chart of open items per engineer
